# مخطط قاعدة البيانات — ملاحظات

مصدر الحقيقة الفعلي لمخطط قاعدة البيانات هو `backend/models.py` (SQLAlchemy ORM).
تُنشأ الجداول تلقائيًا عند أول تشغيل عبر `Base.metadata.create_all()`، أو يدويًا عبر `python seed_data.py`.

## الجداول (21 جدولًا)

| الجدول | الغرض | العلاقات الرئيسية |
|---|---|---|
| `roles` | الأدوار الوظيفية الستة | 1 → N مع `users` |
| `users` | حسابات النظام | ينتمي إلى `roles` |
| `farms` | المزارع | 1 → N مع `zones`, `employees` |
| `zones` | القطاعات الزراعية داخل المزرعة | ينتمي إلى `farms`؛ 1 → N مع `crops`, `trees`, `valves` |
| `crops` | المحاصيل | ينتمي إلى `zones` |
| `trees` | الأشجار (للمحاصيل الدائمة) | ينتمي إلى `zones` |
| `soil_sensors` | حساسات التربة (رطوبة/حرارة) | ينتمي إلى `zones` |
| `weather_sensors` | محطات الطقس | ينتمي إلى `farms` |
| `sensor_readings` | قراءات موحّدة لكل أنواع الحساسات | مرجع عام (`sensor_type` + `sensor_ref_id`) |
| `energy` | لقطات دورية لحالة الطاقة الإجمالية | ينتمي إلى `farms` |
| `solar_panels` | الألواح الشمسية | ينتمي إلى `farms` |
| `batteries` | البطاريات | ينتمي إلى `farms` |
| `water_tanks` | خزانات المياه | ينتمي إلى `farms` |
| `pumps` | المضخات | ينتمي إلى `farms` |
| `valves` | الصمامات | ينتمي إلى `zones` |
| `irrigation_logs` | سجل عمليات الري | يربط `zones` + `pumps` + `users` (من شغّل العملية) |
| `alerts` | التنبيهات | ينتمي إلى `farms`، اختياريًا `zones` |
| `reports` | التقارير المولّدة | ينتمي إلى `farms` |
| `maintenance_tasks` | مهام الصيانة | يربط أصل معدات عام + `users` (المكلَّف) |
| `inventory` | المخزون (قطع غيار، أسمدة...) | ينتمي إلى `farms` |
| `employees` | سجل الموظفين الميدانيين | ينتمي إلى `farms`، اختياريًا `users` |

## ملاحظات تصميمية

- **`sensor_readings` كجدول موحّد**: بدل جدول لكل نوع حساس، يحمل `sensor_type` (نص) و`value` رقمي موحّد. هذا يجعل إضافة نوع حساس جديد (مثل مستشعر أشعة UV) لا يتطلب أي تعديل في قاعدة البيانات.
- **الفهرسة الموصى بها عند الانتقال لـ PostgreSQL**: إضافة فهرس مركّب على `(sensor_type, zone_id, recorded_at)` في `sensor_readings` لتسريع استعلامات لوحة التحكم والتقارير على البيانات الزمنية الكبيرة.
- **جدول تدقيق مستقبلي (`audit_logs`)**: غير موجود في هذه المرحلة، لكنه مُدرج في خارطة الطريق (README) لتسجيل كل عملية حساسة.
